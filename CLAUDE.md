# 📌 CLAUDE SYSTEM RULES - Dividend Note
## [CORE PRODUCT RULE]
이 서비스는 단순 배당 계산기가 아니다.
모든 기능은 아래 구조를 반드시 따라야 한다:
→ 종목 탐색 → 포트폴리오 추가 → 배당 추적 → 반복 방문
다음 구조는 금지한다:
- 검색 → 계산 → 종료
- 단발성 기능
- 페이지 간 연결이 없는 구조
---
## [PRIMARY GOAL]
사용자가 서비스를 반복적으로 방문하도록 만드는 것이 최우선 목표이다.
이를 위해 반드시:
- 시간에 따라 변화하는 데이터 제공
- 사용자 행동(추가, 저장)을 유도
- 개인화된 정보 제공
---
## [DESIGN PRINCIPLES]
1. 모든 기능은 '내 배당(포트폴리오)'으로 연결되어야 한다.
2. 모든 화면은 정적인 정보가 아니라 '변화'를 보여줘야 한다.
   - 예: 전월 대비, D-day, 예상 변화
3. 모든 콘텐츠는 '행동'으로 이어져야 한다.
   - 예: 종목 추가, 포트폴리오 저장
4. 계산 기능은 보조 기능이며, 핵심 기능은 '관리 및 추적'이다.
---
## [SEARCH UX PRINCIPLES]
검색은 단순 입력 기능이 아니라 '탐색 기능'이어야 한다.
반드시 포함:
- 자동완성
- 인기 종목
- 추천 종목
검색 결과는 항상 다음 행동으로 이어져야 한다:
→ 종목 선택 → 포트폴리오 추가
---
## [PORTFOLIO FIRST STRATEGY]
"내 배당" 페이지는 서비스의 중심이다.
모든 기능은 아래로 연결되어야 한다:
- 배당 계산기 → 포트폴리오 추가
- 배당 랭킹 → 포트폴리오 추가
- 배당 캘린더 → 포트폴리오 연결
- 배당 위키 → 종목 추가
---
## [DATA EXPERIENCE RULE]
데이터는 반드시 '변화'를 포함해야 한다.
필수 요소:
- 다음 배당 D-day
- 이번 달 예상 배당
- 전월 대비 변화
- 연간 목표 대비 달성률
---
## [ACTION TRIGGERS]
모든 주요 화면에는 아래 중 하나 이상의 액션이 있어야 한다:
- 포트폴리오 추가
- 종목 저장
- 알림 설정
행동 없는 정보 제공은 금지한다.
---
## [ANTI-PATTERNS]
아래 구조는 절대 구현하지 말 것:
- 계산 후 끝나는 UX
- 클릭 후 다음 행동이 없는 화면
- 데이터 변화가 없는 정적인 페이지
- 종목 조회만 가능한 구조 (추가 기능 없음)
---
## [DECISION CHECK]
새 기능을 구현하기 전 반드시 아래 질문을 통과해야 한다:
1. 이 기능이 포트폴리오 추가로 이어지는가?
2. 이 기능이 재방문 이유를 만드는가?
둘 중 하나라도 NO이면 구현하지 말 것.
---
## [SUCCESS CRITERIA]
기능이 성공했는지 판단 기준:
- 사용자가 하루 1번 이상 다시 들어올 이유가 있는가?
- 종목 추가(포트폴리오)가 자연스럽게 발생하는가?
- 서비스가 단순 계산기가 아닌 '관리 도구'로 인식되는가?
---
## [IMPLEMENTATION PRIORITY]
기능 우선순위:
1순위: 포트폴리오 기능 강화
2순위: 검색 UX 개선
3순위: 추천 및 인사이트
4순위: 계산 기능
계산 기능은 절대 핵심 기능이 아니다.
---
## [FINAL RULE]
모든 기능은 아래 질문을 만족해야 한다:
👉 "이 기능이 사용자를 다시 돌아오게 만드는가?"
NO라면 구현하지 말 것.
